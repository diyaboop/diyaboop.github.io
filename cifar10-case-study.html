Problem

Machine learning evaluation has a reporting problem. A model achieves 81% accuracy and the team declares success. That number is real — but it is an average, and averages hide where the model fails. A classifier that performs at 91% on automobiles and 58% on cats is not an 81% model. It is a model with a 33-percentage-point equity gap between its best and worst-served classes.

The same problem extends beyond bias. A model trained on clean images may collapse under minor real-world perturbations — a slight rotation, gaussian noise, brightness shift — despite performing well in controlled evaluation. And a model that expresses 90%+ confidence on a prediction it gets wrong is not just inaccurate; it is unreliable in a way that damages user trust and downstream decision quality.

The audit gap: Standard model evaluation reports a single accuracy number. It does not tell you which classes are being failed, whether the model survives real-world conditions, or whether its confidence scores mean anything. This project builds the infrastructure to answer all three questions — systematically, reproducibly, and with statistical evidence.

Solution

A three-module responsible AI audit pipeline applied to a CNN trained on CIFAR-10. Each module targets a different dimension of model reliability — fairness, robustness, and calibration — and produces structured, evidence-based findings rather than pass/fail verdicts.

Bias Audit Findings

Per-class accuracy reveals a 32.8 percentage-point gap between the model's best and worst-performing classes. This disparity is statistically significant — not sampling noise — and represents a systematic failure to serve certain image categories equally.

Class Accuracy vs Best Class Assessment
Automobile 91.4% — Best performing
Ship 89.2% -2.2pp Strong
Airplane 88.1% -3.3pp Strong
Truck 86.7% -4.7pp Acceptable
Horse 85.3% -6.1pp Acceptable
Deer 82.9% -8.5pp Acceptable
Frog 79.4% -12.0pp Below average
Bird 74.1% -17.3pp Underperforming
Dog 67.3% -24.1pp Significant gap
Cat 58.6% -32.8pp Worst performing
Z=16.9, p<0.001: The 32.8pp accuracy gap between automobile (91.4%) and cat (58.6%) is statistically significant — not a random fluctuation. The model has learned to classify some categories far more reliably than others. In a deployed system, this means cat images are systematically and predictably misclassified at nearly twice the rate of automobile images.

The pattern is not random. Visually similar classes — cat/dog, deer/horse, bird/airplane — cluster together in confusion matrices, suggesting the model struggles with fine-grained intra-category distinctions. The bias isn't arbitrary; it reflects what the model architecture and training regime made easy versus hard.

Robustness Audit Findings

Real-world images are not clean training samples. They arrive rotated, noisy, differently lit. A model that achieves 81% on a curated test set may fail substantially under conditions that users will routinely encounter. The robustness audit tests exactly this — controlled perturbations applied to the test set, measuring accuracy degradation.

Perturbation Baseline Accuracy Post-Perturbation Drop Assessment
Rotation (15°) 81.17% 69.5% -11.67pp Not rotation-invariant
Gaussian Noise (σ=0.1) 81.17% 72.3% -8.87pp Noise-sensitive
Brightness Shift (+0.2) 81.17% 78.9% -2.27pp Relatively robust
A 15° rotation — barely perceptible to a human — drops accuracy from 81% to 69.5%. This is not a catastrophic failure. It is a predictable one. Any deployment context where images arrive at arbitrary orientations (mobile cameras, surveillance feeds, user uploads) would expose this fragility. The model was not designed to be rotation-invariant, and without augmentation, it isn't.

Gaussian noise sensitivity is the second notable finding. A noise level of σ=0.1 — corresponding to mild image corruption — causes an 8.87pp accuracy drop. This has implications for any real-world deployment where image quality is not controlled: compressed images, low-light captures, or scanned documents would all trigger similar degradation.

Calibration Audit Findings

Calibration measures whether a model's confidence scores mean what they imply. A well-calibrated model that predicts 90% confidence should be correct 90% of the time. An overconfident model expresses high certainty on predictions it gets wrong — which is more dangerous than low accuracy, because it misleads downstream decision-makers.

54% of all predictions are made with >90% confidence. Of those high-confidence predictions, 162 are wrong — predictions where the model expressed near-certainty and was incorrect. This is the calibration failure: not that the model is inaccurate, but that it doesn't know when it's inaccurate.

What This Means in Practice

  • A downstream system that filters on confidence (>90% → act automatically) would act on 162 wrong predictions per evaluation cycle without any human check
  • User-facing confidence displays ("81% sure this is a cat") would systematically mislead users on exactly the cases where the model is most wrong
  • Calibration can be fixed post-hoc with temperature scaling — but only if the problem is measured first. This audit makes it measurable.

Who This Is Built For

ML Engineer

Needs per-class accuracy breakdowns and perturbation results to identify where to invest training effort — more augmentation, class-weighted loss, or architectural changes. The bias audit surfaces exactly which classes need attention and by how much.

Product Manager

Needs to understand whether a model is deployment-ready beyond headline accuracy. The three-module audit provides a structured readiness assessment: is the model fair across categories, stable under real-world conditions, and honest about its uncertainty?

AI Safety Researcher

The calibration audit directly addresses the question of whether model confidence is a reliable signal — a prerequisite for any system that uses model confidence to gate human oversight or automated action.

Compliance / Risk Officer

Needs documented evidence of bias testing and statistical significance before deploying an AI system in a regulated environment. Every finding in this audit includes a Z-statistic and p-value — ready to include in model risk documentation.

Impact & Key Takeaways

  • 32.8pp class accuracy gap — invisible in aggregate reporting, surfaced by per-class analysis with statistical significance (Z=16.9, p<0.001)
  • 15° rotation causes 11.67pp accuracy drop — a model that appears robust on clean data is not rotation-invariant in real-world conditions
  • 162 high-confidence wrong predictions — calibration failure that would silently mislead any downstream system trusting confidence scores
  • Full audit reproducible in one pipeline run — not a one-time evaluation but a repeatable process applicable to any image classifier
The broader point: 81.17% overall accuracy is a starting point, not a verdict. Responsible AI evaluation requires asking three additional questions: which groups are being failed, does the model hold up under realistic conditions, and does it know when it's wrong? This audit operationalizes all three — and provides the statistical scaffolding to turn findings into action.
